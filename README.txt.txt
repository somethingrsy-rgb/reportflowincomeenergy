배포 방법
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Netlify Function은 단순 Drag & Drop 배포만으로는 정상 동작하지 않습니다.
반드시 아래 중 하나를 사용하세요:
  - Netlify CLI 배포: netlify deploy --prod
  - GitHub 저장소 연결 후 자동 배포 (권장)

필수 Netlify 환경변수
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1) FIREBASE_DATABASE_URL
   Firebase 콘솔 > 실시간 데이터베이스 > 데이터베이스 URL
   예: https://<프로젝트ID>-default-rtdb.asia-southeast1.firebasedatabase.app

2) FIREBASE_PROJECT_ID
   Firebase 콘솔 > 프로젝트 설정 > 일반 > 프로젝트 ID

3) FIREBASE_CLIENT_EMAIL
   Firebase 콘솔 > 프로젝트 설정 > 서비스 계정 > 새 비공개 키 생성
   다운로드한 JSON 파일의 client_email 값

4) FIREBASE_PRIVATE_KEY
   위 JSON 파일의 private_key 값
   (줄바꿈 \n 포함하여 통째로 붙여넣기)

5) ADMIN_PIN
   관리자 인증에 사용할 PIN 번호
   ※ 보안상 코드나 문서에 절대 기록하지 마세요.

6) ADMIN_PIN_SECRET (선택)
   토큰 서명용 비밀값. 미설정 시 ADMIN_PIN이 대신 사용됩니다.
   openssl rand -hex 32 로 생성한 긴 랜덤 문자열 권장.

7) ALLOWED_ORIGIN (선택)
   CORS 허용 도메인 및 푸시 알림 클릭 시 이동 링크.
   예: https://mafraincomeenergy.netlify.app

구성 파일
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
index.html
  메인 앱. FCM 토큰 발급/저장, 대기열 UI, 관리자 도구 전부 포함.

service-worker.js
  백그라운드 FCM 푸시 수신 처리.

manifest.webmanifest
  PWA(홈 화면 추가) 설정.

netlify.toml
  Netlify Functions 디렉토리 경로 설정.

package.json
  firebase-admin 의존성 패키지 정의.

netlify/functions/_config.js
  Firebase Admin SDK 초기화 공통 모듈. 모든 함수가 공유.

netlify/functions/verify-pin.js
  PIN 검증 후 서명된 임시 토큰 발급 (유효기간 7일).
  실제 PIN은 Netlify 환경변수에만 존재하며 코드에 없음.

netlify/functions/admin-write.js
  관리자 전용 Firebase 쓰기. 토큰이 유효해야만 저장 허용.
  브라우저 콘솔 조작으로 우회 불가.

netlify/functions/send-next-notification.js
  현재 REPORTING 상태인 대상자 기기로 FCM 푸시 알림 발송.
  보고완료 / 다음호출 시 자동 호출됨.

netlify/functions/notify-deputy.js
  국장님 부재중 전환 시 등록된 대리 수신자 기기로 FCM 푸시 발송.

주요 기능
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[일반 사용자 - 누구나]
  · 보고 예약 등록 / 취소
  · 내 순번 실시간 확인
  · 긴급 보고 신청 (사유 입력 필수, 관리자에게만 공개)
  · 순번 임박 / 입장 알림 (브라우저 알림 + FCM 백그라운드 푸시)

[관리자 도구 - PIN 인증 필요]
  · 다음 순번 호출 / 보고완료 처리
  · 특정 예약 강제 삭제
  · 긴급 지정 / 긴급 해제
  · 전체 대기열 리셋
  · 테스트용 대기열 자동 생성 (시뮬레이션)
  · 국장님 부재중 설정 / 복귀 처리
  · 대리 수신자 기기 등록
  · 부재중 이력 조회 (최대 30건)

[국장님 화면 - PIN 인증 필요, 최초 1회]
  · 대기열 전용 뷰 (예약 양식 숨김, 대기열 크게 표시)
  · 부재중 설정 / 복귀 처리 → PIN 재입력 없이 바로 가능

관리자 인증 방식
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
· "관리자 도구 켜기" / "국장님 화면" 버튼은 앱 시작 시 항상 표시됨
  (이전 버전은 토큰 만료 시 버튼이 사라지는 문제가 있었으나 수정됨)
· 버튼 클릭 → PIN 입력 팝업 → Netlify 서버에서 검증 → 서명된 토큰 발급
· 토큰은 기기 localStorage에 저장, 유효기간 7일
· 토큰 유효 시 재입력 불필요
· 토큰 만료 시 기능 실행 시점에 PIN 팝업 자동 재표시 (alert 없음)
· PIN 변경: Netlify 환경변수 ADMIN_PIN 수정 → 재배포

국장님 부재중 관리
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
· 상단 배너에서 재실/부재중 상태를 누구나 실시간 확인 가능
· 부재중 사유: 출장 / 내부회의 / 외부회의 / 휴식 / 기타
· 메모 및 예상 복귀 시간 입력 가능
· 부재중 중 호출 시 → 자동 예약 상태로 대기
  복귀 처리 시 → 대기 중 다음 순번 자동 호출 + 푸시 알림 발송
· 부재중 전환 즉시 대리 수신자 기기로 FCM 푸시 자동 발송
· 부재중 이력 최대 30건 자동 기록

[대리 수신자 등록 방법]
  1. 담당 기기(비서실 등)에서 앱 접속
  2. "알림 활성화" 버튼 클릭 → FCM 토큰 생성
  3. 관리자 도구 > "이 기기를 대리 수신자로 등록" 클릭

안드로이드 연결 안정성 (이번 버전 수정)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
· 앱 백그라운드 복귀 시 Firebase 즉시 재연결
· 네트워크 전환(와이파이↔LTE) 시 즉시 재연결
· 연결 실패 시 최대 3회 자동 재시도 (Firebase 토큰 강제 갱신 포함)
· 연결 상태 표시 정확화: "실시간 연결됨" / "연결 장애" 구분 표시

배포 후 확인 사항
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. 앱 접속 → 상단 "실시간 연결됨" 표시 확인
2. "관리자 도구 켜기" 클릭 → PIN 입력 팝업 확인
3. PIN 입력 → 관리자 패널 표시 확인
4. 예약 등록 → 대기열에 표시 확인
5. 관리자 > "테스트용 대기열 자동 생성" → 시뮬레이션 확인
6. "알림 활성화" → 호출 시 푸시 알림 수신 확인
7. 안드로이드: 앱 백그라운드 전환 후 복귀 → "실시간 연결됨" 확인
